Project Overview

The BCEM website is a modern Single Page Application (SPA) built with React and Vite. It emphasizes performance, modularity, and a premium "University" aesthetic using Tailwind CSS.

React 18 Vite Tailwind CSS

Installation & Setup

Follow these steps to set up the project locally:

# 1. Clone the repository
git clone https://github.com/iam-neo/BCEM.git
cd BCEM

# 2. Install dependencies (Node.js 18+ required)
npm install

# 3. Start development server
npm run dev

# 4. Build for production
npm run build

Project Structure

The codebase follows a modular structure where each component and page is self-contained.

  • src/
    • components/ - Reusable UI building blocks (Button, Card, etc.)
    • pages/ - Main route views (Home.jsx, About.jsx, etc.)
    • App.jsx - Main layout & route definitions
    • main.jsx - Entry point, renders App into DOM
    • index.css - Global styles & Tailwind directives
  • tailwind.config.cjs - Design tokens (colors, fonts, animations)
  • vercel.json - Server configurations for deployment

Component Logic: Button

The Button component uses a Variant Pattern to be reusable across 6 different styles without code duplication.

How it works:

  • It accepts a variant prop (e.g., 'primary', 'outline').
  • It looks up the CSS classes for that variant in a dictionary object variants.
  • It combines baseClasses (common logic like padding/rounded) with the selected variant classes.

Component Logic: Card

The Card uses the Compound Component Pattern. Instead of one giant component with 50 props, it exposes sub-components (Card.Image, Card.Title) attached to the main Card function.

Backend Architecture (Future Roadmap)

To scale the application for a full College Management System (CMS), we need to transition from a Static App to a Client-Server Architecture.

Why an API is needed?

Even for local testing, the React Frontend (Client) must communicate with the Backend (Server) via an API. This is because React runs in the browser environment (even on localhost) and cannot securely access the database directly.

Decoupled Architecture

  • Frontend (Port 5173): The React App ("The Face"). Calls APIs.
  • Backend (Port 8000/3000): The Logic Server ("The Brain"). Handles Auth, Database, Logic.
  • Database (Port 3306): MySQL/PostgreSQL ("The Memory"). Stores student data.

Role-Based Access Control (RBAC)

To support multiple login types (Student, Teacher, Admin, Exam Dept, Super Admin), we implement RBAC.

Logic Flow:

  1. User logs in via React Form.
  2. Backend verifies credentials and checks the role column in the database.
  3. Backend returns a secure Token (JWT) containing the role (e.g., role: 'TEACHER').
  4. React saves the token and uses "Protected Routes" to allow/deny access to specific pages.

Recommended Backend Stack

Based on the requirements, PHP / Laravel is the recommended backend choice.

Laravel MySQL React

Why Laravel?

  • Built-in Auth: Robust authentication and role management out of the box.
  • API Ready: Excellent support for REST APIs to serve our React frontend.
  • Admin Panels: Ecosystem tools (like Filament) can auto-generate admin dashboards.
  • Security: Industry-standard security features (CSRF, SQL Injection protection).

Deployment Fixes

Initial deployments failed because node_modules (Windows binaries) were uploaded to git. To fix this logic:

  1. Created a .gitignore to exclude node_modules.
  2. Converted config files (tailwind.config.js) to CommonJS (.cjs) for better compatibility with Vercel's build environment.
